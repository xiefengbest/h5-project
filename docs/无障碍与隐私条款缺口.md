# AnySelected H5：无障碍与隐私/条款 — 实现缺口整理

本文档汇总 `pages-mobile/web-anyselected/index.html` 与**无障碍声明**、**隐私政策**、**服务条款**正文之间的差异，便于原型补全。**不构成法律意见**，条款与合规策略需法务确认。

## 一、无障碍（Accessibility）

### 1.1 声明中已承诺、但代码侧仍薄弱或缺失的部分

| 主题 | 声明要点 | 当前实现缺口 | 建议方向 |
|------|----------|--------------|----------|
| 模态对话框（语言选择） | 键盘、ARIA、可访问性 | 无 `Escape` 关闭；打开后未把焦点移入抽屉；关闭后未将焦点回到语言按钮；无焦点陷阱；背后内容未 `inert` / `aria-hidden` 与 `aria-modal` 配套 | 按 WAI-ARIA 模态实践：焦点管理、Escape、陷阱、`inert` 或等价处理 |
| 分类筛选条 | 「适当时使用 ARIA」 | 使用 `role="tablist"` + `role="tab"`，但无对应 `tabpanel`，语义并非标签页 | 改为普通 `button` + `aria-pressed`，或 `role="toolbar"`；若坚持选项卡模型需可见 `tabpanel` |
| 搜索与结果 | 表单/控件可感知 | 结果区仅 `hidden` 切换，输入框与结果区缺少 `aria-controls` / `aria-expanded` 等关联 | 补充关联属性，或采用 Combobox + listbox 模式 |
| 跳到主要内容 | 跳过链接 | 目标为 `#mainContent`，部分环境下焦点未必落在主区域 | 为 `#mainContent` 设 `tabindex="-1"`，必要时在激活 skip 时 `focus()` |
| WCAG 2.1 AA | 声明目标符合 AA | 无自动化检测与走查记录 | CI 跑 axe/Lighthouse、关键路径键盘与读屏人工测；对比度与 200% 缩放需设计/CSS 实测 |
| 动画 | — | 未使用 `prefers-reduced-motion` 减弱动效 | 对过渡/动画做减弱或关闭 |

### 1.2 其他可增强项（非阻塞但常见）

- 顶栏若为整站导航语义，可考虑使用 **`header` + `nav`** 等地标。
- 列表/详情切换若变化复杂，可按需增加 **`aria-live`** 简报（避免过度播报）。

### 1.3 优先级建议

1. **高**：语言抽屉（焦点、Escape、陷阱、背后不可交互）。
2. **高**：分类条 ARIA 模型修正（避免误用 `tab`）。
3. **中**：搜索与结果区的可访问关联；skip 链接与 `main` 焦点。
4. **中低**：减少动效、`aria-live`、地标结构优化；建立 AA 检测与走查流程。

## 二、隐私政策（Privacy Policy）

### 2.1 正文已写、但产品/代码尚未对齐的部分

| 主题 | 政策表述要点 | 当前缺口 | 建议 |
|------|--------------|----------|------|
| Cookie 同意 | 分析/广告 Cookie 仅在用户通过 **Cookie 横幅**同意后收集 | 无 Cookie 条、无同意状态存储与逻辑 | 接入 CMP 或自研横幅；非必要脚本须 **同意后**再加载 |
| Google Analytics / Ad Manager | 明确使用 GA、GAM、CDN | 页面内未见 gtag/GTM/GPT 等（若未上线应改文案） | 若上线：配置 **Consent Mode**（或等价）与同意状态一致 |
| 撤回同意 | 用户可撤回 Cookie/广告相关同意 | 无「Cookie 设置」等入口 | 与 CMP 或偏好中心绑定常驻入口 |
| 政策更新 | 重大变更将告知、必要时重新征求同意 | 页内无「最后更新」日期/版本 | 至少增加 **生效日或版本号**；重大变更可用横幅提示 |
| 本地存储 | — | 代码使用 `localStorage`（如语言键） | 在隐私政策中简要说明用途与是否构成个人数据 |

### 2.2 可选（视辖区与业务）

- 分包商列表、DPA、欧盟代表、CCPA「Do Not Sell」等。
- WebView 场景下与原生端隐私说明的**一致性与链接**。
