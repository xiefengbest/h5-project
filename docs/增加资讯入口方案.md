# 增加资讯入口方案

## 背景与核心洞察

该 H5 模拟 Android 壁纸 App 交互，关键特性：

1. 详情页支持从**锁屏状态左滑直接进入**，这是全站最大流量入口
2. 进入详情页后，**主图区域支持继续左滑**切换下一张壁纸（顶部主图轮播，底部推荐网格随之刷新）
3. **Banner 广告计划全部移除**，腾出空间

用户最高频路径：

> 锁屏左滑 -> 详情页主图 -> 继续左滑浏览更多壁纸 -> (Save / 全屏预览 / 返回)

左滑轮播是用户停留时间最长、交互频率最高的行为，这里是资讯入口的最佳切入点。

## 改造前问题

- 资讯入口仅有主图底部小字文案叠层一种形态，视觉权重低
- 文案随机分配，与图片无关，缺乏点击动机
- Banner 广告占据黄金位置，但计划移除，操作栏与推荐网格之间将出现空白

## 用户路径与入口机会

**用户怎么进到详情页、在详情页会干啥：**

- 从**锁屏左滑**或从**首页点某张图**，都会打开详情页（当前大图）。
- 在详情页里，用户会：**左滑换图**、**点 Save**、**全屏预览**、**返回**，或者**往下滚**看到下面的推荐壁纸。

**资讯可以出现在哪些地方（已做 / 可做）：**

- **入口1（已做）**：左滑看主图时，轮播里每隔几张就插一张「资讯大卡」，滑到就看见。
- **入口2（已做）**：原来 Banner 广告那一块改成「横滑资讯推荐区」，往下滚就能看到。
- **入口4（已做）**：下面推荐壁纸的网格里，固定几个位置放资讯卡片，和壁纸混排。
- **入口3、5、6、7、8（未做）**：例如 Save 后弹层、全屏预览上滑、返回时挽留卡、首页瀑布流里插资讯卡、浮动气泡等，后续可按需加。

+-------------------------------+

## 需求描述（三个资讯入口）

以下对详情页已实施的三个资讯入口做产品级需求描述，便于评审、排期与后续服务端联调。

### 1. 主图资讯大卡（轮播流资讯卡）

| 项目 | 说明 |
|------|------|
| **功能概述** | 在详情页主图左滑轮播流中，按固定间隔插入全屏资讯大卡，与壁纸主图同尺寸展示，左滑即可刷到。 |
| **展示位置** | 主图轮播流内，与壁纸项交替出现；由 Firebase 配置：positions 数组指定第几个位置出资讯卡（如 [4, 8, 12]）。 |
| **展示形态** | 与主图同比例（如 9:16）的全屏卡片：渐变背景、顶部「热点资讯」标签、主标题、简短摘要、底部「查看详情」按钮；无壁纸图片。 |
| **交互规则** | 轮播到资讯卡时隐藏主图操作栏（爱心、Save）；左/右滑可切换上一张/下一张（壁纸或资讯）；点击「查看详情」跳转至该条资讯落地页。 |
| **数据与配置** | 资讯内容、插入间隔、是否启用均由 **Firebase** 下发（见Firebase 资讯位配置）；支持 A/B 与灰度。 |

### 2. 横滑资讯卡片（资讯推荐区）

| 项目 | 说明 |
|------|------|
| **功能概述** | 在详情页操作栏下方、原 Banner 位，展示「热点资讯」横滑列表，用户左右滑动浏览多条资讯入口。 |
| **展示位置** | 操作栏与下方「推荐」壁纸网格之间，单行横滑区域；与主图轮播流、推荐网格均可见（首屏内或稍下滚动）。 |
| **展示形态** | 区域标题「热点资讯」+ 右侧「更多」链接；横向滚动的卡片列表：每卡含缩略图、标题，卡片宽度固定（如 200px），支持横向吸附滚动（scroll-snap）。 |
| **交互规则** | 横向滑动浏览卡片；点击单卡跳转对应资讯页；点击「更多」进入资讯列表/频道页。 |
| **数据与配置** | 仅是否展示由 **Firebase** 配置（section.enabled 开关，见Firebase 资讯位配置；展示条数与内容由新闻接口或前端约定。 |

### 3. 推荐网格中的资讯卡

| 项目 | 说明 |
|------|------|
| **功能概述** | 在详情页底部「推荐」壁纸网格中，在固定位置插入一张资讯专属卡片，与壁纸卡片、广告卡并列展示。 |
| **展示位置** | 「推荐」标题下的两列网格内，由 Firebase 配置：positions 数组指定第几个格子出资讯卡（如 [4, 7]）。 |
| **展示形态** | 与壁纸卡同栅格、同圆角，但内容形态区分：渐变背景 +「热点资讯」标签 + 标题 +「阅读全文」类 CTA，无缩略图，视觉上明显区别于壁纸卡。 |
| **交互规则** | 点击整张资讯卡跳转至该条资讯落地页；与壁纸卡点击（进详情/预览）行为分离。 |
| **数据与配置** | 是否展示、第几个位置出由 **Firebase** 的 positions 配置见 Firebase 资讯位配置；展示内容由新闻接口获取。 |

## Firebase 资讯位配置（仅位置）

上述三个资讯入口的**展示位置、是否启用、插入间隔**等在 **Firebase** 中配置；**新闻内容由单独接口获取**，不与位置配置混在一起。客户端拉取 Firebase 位置配置后，再请求新闻接口，组合渲染，无需发版即可调整展示位。

本配置**只包含三个资讯入口的位置与开关**，不包含具体新闻列表。配置存储建议使用 **Firestore** 单文档（如 `config/newsSlot`）或 **Remote Config** 参数（JSON 字符串）。

### 配置示例（复制到 Firestore / Remote Config）

```json
{
  "carousel": {
    "enabled": true,
    "positions": [4, 8, 12]
  },
  "section": {
    "enabled": true
  },
  "grid": {
    "enabled": true,
    "positions": [4, 7]
  }
}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| **carousel** | 主图轮播流资讯大卡 |
| carousel.enabled | 是否在轮播中插入资讯卡 |
| carousel.positions | 第几个位置出资讯卡（1-based 数组，如 [4, 8, 12] 表示第 4、8、12 个位置为资讯卡） |
| **section** | 横滑资讯推荐区 |
| section.enabled | 是否展示横滑资讯区（仅开关，条数由前端或接口约定） |
| **grid** | 推荐网格中的资讯卡 |
| grid.enabled | 是否在推荐网格中展示资讯卡 |
| grid.positions | 第几个格子出资讯卡（1-based 数组，如 [4, 7] 表示第 4、7 个格子为资讯卡） |
